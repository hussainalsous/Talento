توثيق الكنترولرات — TALENTO

توثيق كنترولرات منصة TALENTO

هذا المستند يوثّق كل كنترولر (Controller) داخل مجلد app/Http/Controllers في الواجهة البرمجية (API v1) لمنصة TALENTO للتوظيف، المبنية على إطار عمل Laravel. التوثيق مستخرَج من الكود الفعلي لكل دالة، ويشمل الغرض، المعاملات، القيمة المرجعة، الـ Route المرتبط، قواعد التحقق (Validation)، والخدمات والموديلات والموارد المستخدمة.

إصدار الـ API (البادئة /api/v1)

ملاحظات معمارية عامة

  • معظم الكنترولرات نحيفة (Thin) وتفوّض منطق العمل إلى طبقة الخدمات (Services).
  • التحقق من المدخلات يتم عبر Form Requests منفصلة تحت app/Http/Requests.
  • صياغة الاستجابات تتم عبر API Resources تحت app/Http/Resources.
  • التخويل (Authorization) يتم عبر Policies باستخدام $this->authorize() أو عبر Middleware للأدوار.
  • التخزين المؤقت (Caching) يتم عبر RedisCacheService في النقاط العامة عالية القراءة.
  • جميع الكنترولرات (عدا EmailVerificationController) ترث من الكنترولر الأساسي App\Http\Controllers\Controller.

Controller — الكنترولر الأساسي

Base / abstract

app/Http/Controllers/Controller.php

كلاس مجرّد (abstract) يُعدّ الأساس لجميع كنترولرات التطبيق. لا يحتوي على دوال مباشرة، لكنه يستخدم الـ trait المسمّى AuthorizesRequests من Laravel، ما يوفّر لكل كنترولر يرث منه إمكانية استدعاء $this->authorize() و $this->authorizeForUser() لتطبيق سياسات التخويل (Policies).

ملاحظة: كنترولر EmailVerificationController هو الاستثناء الوحيد — فهو يرث من Illuminate\Routing\Controller مباشرةً (وليس من الكلاس الأساسي للمشروع) لأنه لا يحتاج إلى قدرات التخويل.

JobPostController

عام / Public extends Controller

app/Http/Controllers/Api/V1/JobPostController.php

نظرة عامة

عرض إعلانات الوظائف المنشورة لعامة المستخدمين (سواء مسجّلين الدخول أم لا). يعتمد على RedisCacheService لتخزين نتائج القوائم المفلترة مؤقتاً لمدة 300 ثانية (5 دقائق) عبر الثابت TTL.

الموديلات: JobPost
الخدمات: RedisCacheService
الموارد (Resources): JobPostResource
Middleware: لا يوجد (نقاط عامة)
GET /api/v1/job-posts · index()

يعرض قائمة مُصفّحة (paginated) لإعلانات الوظائف المنشورة فقط (JobPost::published()). يبني مفتاح كاش من الفلاتر؛ فإن وُجدت نسخة مخزّنة يعيدها مباشرة، وإلا ينفّذ الاستعلام ثم يخزّن النتيجة. يدعم فلاتر البحث النصي في العنوان والوصف، والموقع، ونوع التوظيف، والمهارات، والحد الأدنى/الأقصى للراتب، بالإضافة إلى الفرز عبر حقول مسموح بها (created_at, salary_min, salary_max, expires_at).

المعاملات:

  • Request $request — معاملات الاستعلام (Query): search, location, employment_type, skill_ids, salary_min, salary_max, sort, dir, page, per_page (الحد الأقصى لـ per_page هو 50).

القيمة المرجعة: JsonResponse تحتوي على success و data (مجموعة JobPostResource) و meta (بيانات الترقيم).

التحقق: لا يوجد Form Request — تُقرأ الفلاتر مباشرة مع تطهير محدود (الحد الأقصى للصفحة + قائمة فرز بيضاء).

GET /api/v1/job-posts/{jobPost} · show()

يعرض تفاصيل إعلان وظيفة واحد عبر Route Model Binding. يرفض الطلب بـ 404 إذا لم تكن حالة الإعلان published (عبر abort_unless)، ثم يحمّل العلاقات company, skills, jobTitle.

المعاملات:

  • JobPost $jobPost — يُحقن تلقائياً من معرّف الـ URL.

القيمة المرجعة: JsonResponse تحتوي على success و data (JobPostResource واحد).

المعاملات:

المعاملات:

  • Request $request — limit (افتراضي 10).

القيمة المرجعة: JsonResponse مع data (قائمة CourseResource).

EmailVerificationController

روابط موقّعة (Signed) extends Illuminate\Routing\Controller

app/Http/Controllers/Api/V1/Auth/EmailVerificationController.php

نظرة عامة

التحقق من البريد الإلكتروني. يرث استثنائياً من Illuminate\Routing\Controller وليس من الكنترولر الأساسي للمشروع (لا يحتاج التخويل). يعتمد على توقيع HMAC للرابط الموقّع (hasValidSignature()) كبديل عن المصادقة. يدعم إعادة التوجيه إلى واجهة React عند ضبط APP_FRONTEND_URL، وإلا يعيد JSON (لدعم عملاء API مثل Flutter) عبر الدالة الخاصة respond().

الموديلات: User, CompanyRegistrationRequest
Middleware: verify/verify-registration بلا حماية (التوقيع هو الاعتماد)؛ resend ضمن auth:sanctum + throttle:6,1
GET /api/v1/email/verify/{id}/{hash} · verify()

يتحقق من صحة الرابط الموقّع ويعلّم بريد المستخدم كمُتحقَّق منه. يميّز بين الرابط المنتهي والمُلاعَب به لرسائل أوضح، ويتحقق من تطابق الـ hash مع sha1(email). يعالج حالات: توقيع غير صالح (403)، مستخدم غير موجود (404)، رابط غير صالح (403)، بريد متحقَّق مسبقاً (200)، أو نجاح التحقق (200).

المعاملات: Request $request, int $id, string $hash.

المرجعة: JsonResponse | RedirectResponse.

GET /api/v1/email/verify-registration/{id}/{hash} · verifyRegistration()

يتحقق من بريد طلب تسجيل الشركة قبل مراجعة الأدمن. يطابق الـ hash مع sha1(requester_email)، ويضبط email_verified_at على الطلب، لينتقل هذا التحقق لاحقاً إلى حساب المستخدم عند الموافقة.

المعاملات: Request $request, int $id, string $hash.

المرجعة: JsonResponse | RedirectResponse.

POST /api/v1/email/verification-notification · resend()

يعيد إرسال بريد التحقق للمستخدم الحالي. يعيد دائماً JSON. إن كان البريد متحقَّقاً مسبقاً يعيد رسالة بذلك دون إرسال. محمي بـ auth:sanctum + throttle:6,1.

المعاملات: Request $request. المرجعة: JsonResponse.
