Job Portal API

1. Authentication

This API uses Laravel Sanctum for stateless Bearer token authentication. After login or registration, include the token in every protected request.

Required Headers
Authorization: Bearer {your_token_here}
Accept:        application/json
Content-Type:  application/json
Example Authenticated Request
GET /api/v1/auth/me HTTP/1.1
Authorization: Bearer 3|abc123xyz...
Accept:        application/json
ℹ️
On logout, the token is revoked server-side. On deactivation, all tokens are revoked immediately.
📦

  "success": false,
  "message": "Unauthorized."
}

11. Example Full Flows

Flow A — Job Seeker: Register → Upload CV → Apply

Step 1 — Register

{
  "first_name": "John", "last_name": "Doe",
  "email": "john@example.com",
  "password": "secret123",
  "password_confirmation": "secret123"
}
→ Save data.token
Step 2 — Upload CV (multipart/form-data)
POST /api/v1/job-seeker/cvs
Authorization: Bearer {token}
title=My 2024 CV  |  file=@cv.pdf  |  is_primary=1
Step 3 — Apply
POST /api/v1/job-posts/5/apply
Authorization: Bearer {token}
{ "cv_id": 3, "cover_letter": "I am excited..." }

Flow B — Company: Register → Admin Approves → Post Job

Step 1 — Submit Registration
POST /api/v1/auth/company-registration-requests
{ "company_name": "Acme Corp", "requester_email": "alice@acme.com", ... }
→ Awaits admin approval
Step 2 — Admin Approves
PATCH /api/v1/admin/company-registration-requests/5/approve
Authorization: Bearer {admin_token}
→ Creates User + CompanyMember + Company in one DB transaction
Step 3 — Post Job
POST /api/v1/company/job-posts
Authorization: Bearer {company_token}
{
  "title": "Senior JS Dev",
  "employment_type": "full_time",
  "status": "published"

12. Observations & Recommendations
🔴
No duplicate application guard. Applying twice to the same job will throw a DB constraint 500. Fix: add pre-check in the service or unique rule in ApplyJobRequest.
🟡
role_in_company has no enum validation. Accepts any free-form string. Consider adding in:company_owner,company_member to CreateMemberRequest.
🟡
No duplicate invitation check. A company can send multiple invitations to the same candidate for the same job. Add unique constraint on (company_id, job_seeker_id, job_post_id).
🟡
Withdraw status guard. Ensure the policy returns a clean 422 when withdrawing an already accepted/rejected application.
🔵
File uploads require multipart/form-data — not application/json. Affected: POST /company/logo, POST /job-seeker/cvs, PATCH /job-seeker/cvs/{id}, POST /auth/company-registration-requests (with logo).
🔵
PUT /job-seeker/skills is a full replace. The only PUT in the API. Frontend must always send the complete skill array — partial sends will remove existing skills.
