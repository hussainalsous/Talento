السياسات والخدمات ومزوّدو الخدمة

يوثّق هذا المستند طبقات منطق الأعمال والتخويل والإعداد في منصة TALENTO للتوظيف (Laravel): السياسات (Policies) التي تحكم التخويل لكل موديل، الخدمات (Services) التي تحوي منطق الأعمال وتُحقَن في الكنترولرز، ومزوّدي الخدمة (Service Providers) الذين يربطون ويُهيّئون التطبيق.

ملاحظات معمارية عامة

  • تسجيل السياسات: لا يوجد AuthServiceProvider ولا تسجيل يدوي؛ تعتمد السياسات على الاكتشاف التلقائي (Auto-discovery) في Laravel — كل App\Models\X يرتبط تلقائياً بـ App\Policies\XPolicy (ما عدا CV الذي يطابق CvPolicy).
  • المزوّد الوحيد المسجّل في bootstrap/providers.php هو AppServiceProvider.
  • الخدمتان SupabaseNotificationService و RedisCacheService مسجّلتان كـ Singleton وتُحقنان في معظم الخدمات الأخرى.
  • الكنترولرز نحيفة وتفوّض منطق الأعمال إلى هذه الخدمات عبر حقن المُنشئ (Constructor Injection).
  • الإشعارات تُكتب في قاعدة بيانات Supabase منفصلة (اتصال supabase)، والبريد يُرسَل عبر مهمة الطابور SendEmail.

① السياسات (Policies)

طبقة التخويل لكل موديل — تُستدعى عبر $this->authorize() في الكنترولرز. كل دالة تُرجع bool.

CompanyPolicy

Model: Company

app/Policies/CompanyPolicy.php

تحكم الوصول إلى الشركة وملفها وأعضائها. دالة خاصة مساعدة isMemberOf() تتحقق من انتماء المستخدم لأعضاء الشركة عبر استعلام members()->where('user_id', ...)->exists().

الاستخدام: Company\ProfileController (view, update)، Company\MemberController (view, manageMembers)، Company\JobPostController@store (createJobPost).

شرطيview(User, Company): bool

يسمح إذا كان المستخدم أدمن، أو مالك الشركة (id === owner_user_id)، أو عضواً فيها.

شرطيupdate(User, Company): bool

يسمح فقط إذا كان المستخدم أدمن أو مالك الشركة (الأعضاء لا يستطيعون التعديل).

شرطيapprove(User): bool

يسمح فقط للأدمن (isAdmin()) — اعتماد/رفض الشركات. لا يأخذ معامل Company.

شرطيmanageMembers(User, Company): bool

يسمح للأدمن أو مالك الشركة فقط — إدارة الأعضاء.

شرطيcreateJobPost(User, Company): bool

يسمح للأدمن أو المالك أو أي عضو في الشركة — نشر الوظائف.

JobPostPolicy

Model: JobPost

app/Policies/JobPostPolicy.php

تحكم الوصول إلى إعلانات الوظائف. الدالة الخاصة isCompanyStaff() تتحقق من أن المستخدم عضو شركة وأن companyMember.company_id يطابق jobPost.company_id. الدوال الأربع متطابقة في الشرط.

الاستخدام: Company\JobPostController (view, update, delete, viewApplicants).

شرطيview / update / delete / viewApplicants (User, JobPost): bool

جميعها تسمح إذا كان المستخدم أدمن أو من طاقم الشركة المالكة للإعلان (نفس company_id).

JobApplicationPolicy

Model: JobApplication

app/Policies/JobApplicationPolicy.php

تحكم عرض طلبات التوظيف وسحبها وتحديث حالتها — بمنطق يختلف حسب دور المستخدم (باحث عمل / طاقم شركة / أدمن).

الاستخدام: JobSeeker\ApplicationController (view, withdraw)، Company\JobPostController@updateApplicationStatus (updateStatus).

شرطي حسب الدورview(User, JobApplication): bool

الأدمن: دائماً. باحث العمل: فقط إذا كان صاحب الطلب (jobSeeker?->id === job_seeker_id). طاقم الشركة: إذا كان companyMember?->company_id يطابق شركة الإعلان (application->jobPost->company_id).

شرطيwithdraw(User, JobApplication): bool

يسمح فقط لباحث العمل صاحب الطلب وبشرط أن الحالة ليست withdrawn أصلاً.

شرطيupdateStatus(User, JobApplication): bool

الأدمن: دائماً. وإلا يجب أن يكون عضو شركة (isCompanyMember()) ينتمي لنفس شركة الإعلان.

CvPolicy

Model: CV

app/Policies/CvPolicy.php

تحكم الوصول إلى السير الذاتية مع مراعاة مستوى الظهور (visibility).

الاستخدام: JobSeeker\CvController (update, delete, analyze). دالة view معرّفة لاستخدام طبقة العرض/البحث.

شرطي حسب الظهورview(User, CV): bool

الأدمن: دائماً. صاحب السيرة (jobSeeker?->id === job_seeker_id): دائماً. عضو الشركة: فقط إذا كانت visibility === 'public'. غير ذلك: رفض.

شرطيupdate(User, CV): bool

يسمح للأدمن أو صاحب السيرة فقط.

شرطيdelete(User, CV): bool

يسمح للأدمن (لحذف المحتوى غير اللائق) أو صاحب السيرة.

شرطيanalyze(User, CV): bool

يسمح فقط لصاحب السيرة (حتى الأدمن لا يحلّل سيرة غيره عبر هذه الدالة).

InvitationPolicy

Model: Invitation

app/Policies/InvitationPolicy.php

تحكم الردّ على الدعوات وعرضها. التحقق من كون الدعوة «معلّقة» (pending) يُفرَض في طبقة الخدمة (يعيد 422)، وليس هنا.

الاستخدام: JobSeeker\InvitationController@respond (respond). دالة view معرّفة لطبقة العرض.

شرطيrespond(User, Invitation): bool

يسمح فقط لباحث العمل المُرسَلة إليه الدعوة (jobSeeker?->id === invitation->job_seeker_id).

شرطي حسب الدورview(User, Invitation): bool

الأدمن: دائماً. باحث العمل: إذا كان المدعوّ. طاقم الشركة: إذا كانت الدعوة صادرة من شركته (companyMember?->company_id === invitation->company_id).

UserPolicy

Model: User

app/Policies/UserPolicy.php

تحكم إدارة المستخدمين وعرضهم. عملياً تعتمد مسارات الأدمن أيضاً على middleware role:admin كطبقة حماية أساسية، وهذه السياسة متاحة للاستخدام البرمجي.

شرطيmanage(User $currentUser, User $targetUser): bool

يسمح فقط إذا كان المستخدم الحالي أدمن.

شرطيview(User $currentUser, User $targetUser): bool

يسمح إذا كان أدمن أو يعرض ملفه الشخصي نفسه (id === targetUser->id).

② الخدمات (Services)

طبقة منطق الأعمال — تُحقَن في الكنترولرز عبر المُنشئ. تتعامل مع الموديلات والكاش والإشعارات ومهام الطابور.

AuthService

app/Services/AuthService.php

منطق المصادقة وإدارة الحساب: التسجيل، الدخول مع حماية ضد التعداد والتخمين، الخروج، تغيير/إعادة تعيين كلمة المرور.

Dependencies: SupabaseNotificationService، RedisCacheService
يُستخدم في: Auth\AuthController
أمان: ثابت DUMMY_HASH لمعادلة زمن الاستجابة (timing guard)، وحدّ محاولات عبر RateLimiter (5 محاولات / 900 ثانية)، ورسائل فشل موحّدة لمنع تعداد الحسابات.
methodregisterJobSeeker(array $data): array

داخل معاملة DB: ينشئ User (دور job_seeker) + JobSeeker + توكن Sanctum. ثم يُبطل كاش admin:users:*، يُنشئ إشعار ترحيب، ويُرسل بريد التحقق. يعيد user/jobSeeker/token.

methodsubmitCompanyRegistrationRequest(array $data, ?string $logoPath): CompanyRegistrationRequest

ينشئ طلب تسجيل شركة (كلمة مرور مُجزّأة، حالة pending)، يُبطل كاش الطلبات، يُشعر جميع الأدمن، ويُرسل بريدين عبر SendEmail: تأكيد الاستلام + رابط تحقق موقّع صالح 48 ساعة (URL::temporarySignedRoute).

methodlogin(string $email, string $password, string $ip): array

يفحص حدّ المحاولات (يعيد 429 عند التجاوز)، يستدعي Hash::check دائماً (حتى لمستخدم غير موجود) عبر DUMMY_HASH. يحدّد سبب الفشل داخلياً (user_not_found / invalid_credentials / inactive_user) ويسجّله، لكنه يرمي رسالة موحّدة. عند النجاح يصفّر العدّاد، يحدّث last_login_at، ويصدر توكناً.

methodlogout(User $user): void

يحذف التوكن الحالي (currentAccessToken()->delete()).

methodupdatePassword(User $user, string $newPassword): void

يحدّث كلمة المرور، يبطل جميع التوكنات عدا الحالية، ويُنشئ إشعار «Password Changed».

methodsendPasswordResetLink(string $email): void

يتجاهل بصمت المستخدمين غير الموجودين/غير النشطين (مكافحة التعداد). يولّد توكناً عبر Password::broker()، يبني رابطاً للواجهة، ويُرسل بريد إعادة التعيين عبر SendEmail.

methodresetPassword(array $credentials): void

يتحقق من التوكن عبر Password::broker()->reset()، يضبط كلمة المرور، يحذف جميع التوكنات، يُنشئ إشعاراً، ويُطلق حدث PasswordReset. يرمي ValidationException عند فشل التوكن.
